2.3 CSFLE Implementation
1. Add CSFLE Configuration Parameters. Add to MongoSinkConfig.java:
   * csfle.enabled - Enable/disable CSFLE
   * csfle.key.vault.namespace - Key vault collection namespace
   * csfle.local.cmk - Local Customer Master Key (base64 encoded)
   * csfle.schema.map - JSON schema map for encryption
2. Modify createMongoClient() Method. In MongoSinkTask.java, extend the builder:
        
private static MongoClient createMongoClient(final MongoSinkConfig sinkConfig) {
    MongoClientSettings.Builder builder = MongoClientSettings.builder()
        .applyConnectionString(sinkConfig.getConnectionString())
        .applyToSslSettings(sslBuilder -> setupSsl(sslBuilder, sinkConfig));
    
    // Add CSFLE configuration
    if (sinkConfig.isCsfleEnabled()) {
        builder.autoEncryptionSettings(createAutoEncryptionSettings(sinkConfig));
    }
    
    // ... rest of the existing code
}


private static AutoEncryptionSettings createAutoEncryptionSettings(final MongoSinkConfig config) {
    // Build AutoEncryptionSettings with local CMK
    Map<String, Object> extraOptions = new HashMap<>();
    extraOptions.put("mongocryptdBypassSpawn", true); // If using shared library
    
    return AutoEncryptionSettings.builder()
        .keyVaultNamespace(config.getString("csfle.key.vault.namespace"))
        .kmsProviders(Map.of("local", Map.of("key", config.getLocalCmk())))
        .schemaMap(config.getCsfleSchemaMap())
        .extraOptions(extraOptions)
        .build();
}

// HashTransformer.java
public class HashTransformer implements FieldTransformer {
    @Override
    public BsonValue transform(String fieldName, BsonValue value) {
        if (value.isString()) {
            String hash = DigestUtils.sha256Hex(value.asString().getValue());
            return new BsonString(hash);
        }
        return value;
    }
}

4. Enhanced WriteModelStrategy (NEW: TransformingWriteModelStrategy.java)


public class TransformingWriteModelStrategy extends WriteModelStrategy {
    
    private final Map<String, FieldTransformer> transformers;
    private final WriteModelStrategy delegate;
    private final boolean failOnError;
    
    @Override
    public WriteModel<BsonDocument> createWriteModel(SinkDocument document) {
        // Apply transformations to document
        BsonDocument transformed = applyTransformations(document.getValueDoc().orElse(null));
        
        // Delegate to underlying strategy
        SinkDocument transformedDoc = document.withValueDoc(transformed);
        return delegate.createWriteModel(transformedDoc);
    }
    
    private BsonDocument applyTransformations(BsonDocument doc) {
        if (doc == null) return null;
        
        BsonDocument result = new BsonDocument();
        
        for (Map.Entry<String, BsonValue> entry : doc.entrySet()) {
            String fieldName = entry.getKey();
            BsonValue value = entry.getValue();
            
            // Check if transformation needed
            if (transformers.containsKey(fieldName)) {
                try {
                    value = transformers.get(fieldName).transform(fieldName, value);
                } catch (Exception e) {
                    if (failOnError) {
                        throw new DataException("Failed to transform field: " + fieldName, e);
                    }
                    LOGGER.warn("Transformation failed for field: {}", fieldName, e);
                }
            }
            
            // Handle nested documents recursively
            if (value.isDocument()) {
                value = applyTransformations(value.asDocument());
            } else if (value.isArray()) {
                value = transformArray(value.asArray());
            }
            
            result.put(fieldName, value);
        }
        
        return result;
    }
}

3.3 Evaluation


Complexity Level: 🟡 Medium (6/10)
Risk Level: 🟡 Medium (5/10)
Estimated Effort: 2-3 weeks (development + testing + documentation)


Pros ✅
* Minimal core changes: Leverages existing WriteModelStrategy pattern
* Testable: Easy to unit test transformers independently
* Performance: Only processes configured fields
* Keep transformations MongoDB-aware (BSON types)
* Integrate seamlessly with CSFLE (transform before encryption) (voir Req #1)
* Leverage existing strategy pattern
* Allow complex transformations with MongoDB context
* Maintains CSFLE compatibility: Transformations happen before encryption


Cons ⚠️
* Requires understanding of WriteModelStrategy
* Additional configuration complexity
* Potential performance overhead on large documents
